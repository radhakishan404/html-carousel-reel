04 / 08
Stack map what each layer does
Simple mental model

Think of the agent stack in four layers

Input layer
markitdown helps convert messy files into Markdown so agents can actually reason over them.
Harness layer
Archon is about deterministic, testable AI coding flows instead of ad-hoc prompting.
Agent layer
hermes-agent and rowboat represent execution, memory, and autonomous task work.
Team layer
multica and Twill frame agents as teammates that accept tasks and return artifacts.
Best takeaway / different repos are solving different layers of the same problem
05 / 08
Reality check still true
Do not skip this

Agents are useful now. They are still not magic.

◎
Specs still matter
Bad task definitions still create bad output, just faster and at larger scale.
✓
Review is non-negotiable
HN kernel discussion is a reminder: trust comes from review loops, not from vibes.
∞
Memory can drift
Long-running agents need context discipline, guardrails, and resets.
$
Ops cost is real
Retries, model usage, tool integrations, and debugging overhead all add up.
Shortcut / treat agents like junior automation with better language, not perfect engineers
